Architecture

Five stages.
Every receipt.

Every image passes through all five stages in sequence. Each stage is isolated — failure in one falls back gracefully rather than crashing the pipeline.

◆
01 / INTAKE
Intake Gate
  • Validates JPEG, PNG, WEBP, PDF, HEIC
  • Detects corruption before processing begins
  • Enforces the 10MB size limit early
  • Classifies document type — receipt or invoice
  • Rejects unreadable files with clear diagnostics
◈
02 / OPTIMISE
Adaptive Optimisation
  • Measures brightness, contrast, sharpness, noise
  • Detects skew angle and perspective distortion
  • Only applies operations that are actually needed
  • Verifies each step — rolls back if quality regresses
  • Original image preserved throughout as fallback
◎
03 / OCR
Text Extraction
  • Tesseract LSTM — PSM 6 uniform block mode
  • Multi-pass retry with alternative PSM modes
  • Per-word confidence scores tracked throughout
  • Falls back to original image if optimised fails
  • English, Sinhala, Tamil, Arabic + 5 more scripts
◇
04 / EXTRACT
Universal Extraction
  • Normalises Persian, Sinhala, Tamil, Chinese digits
  • Multi-signal amount scoring — keyword, position, magnitude
  • Date extraction across all major formats worldwide
  • Merchant detection with Levenshtein fuzzy matching
  • 14 universal category types with three-signal scoring
◉
05 / VALIDATE
Validation Gate
  • Field-level validation — range, format, confidence
  • Cross-field consistency checks with penalty modifiers
  • Weighted confidence: amount 40%, date 30%, rest 30%
  • HIGH / MEDIUM / LOW / REJECTED response levels
  • When uncertain returns null — never guesses

Speed
<400ms
No cloud. No round-trips.
Tesseract and OpenCV run entirely inside the container. No external OCR API calls — the pipeline starts and ends on the same machine.
Privacy
Private by architecture
No images stored after processing. No user data retained between requests. Audit logs auto-purge after 24 hours.
Languages
8+
English, Sinhala, Tamil, Arabic, Chinese, French, German, Spanish. Digit normalisation for Persian, Devanagari, Thai scripts.
Confidence
Honest output
Every field returns a score 0–1. When uncertain, returns null — never guesses. A blank field beats a wrong number.
